Typography Hierarchy

Haas Grotesk (or Inter fallback). Emphasis comes from size and color, never from weight. Display type is always 400, never bold.

40px Display · Weight 400
Page h1 · Hero headline
32px Display · Weight 400

24px Display · Weight 400

11px Label · Weight 600
Metric label, uppercase
14px Body · Weight 400
Running text, default
13px Caption · Weight 500
Meta text, captions
  • Never use weight 700 or 800. Display is 400, buttons/labels are 500.
  • 14px is the default body size. Never smaller than 13px in interface.
  • Emphasis comes from color contrast, not weight. Use signature colors to highlight.

Before & After: Key Changes

❌ Before (Current)

  • Purple accent buttons
  • Mixed typography weights
  • Dense layout, no breathing room
  • 10+ elements per viewport
  • No narrative structure
  • Chaotic color palette

✓ After (New)

  • Near-black primary CTA
  • Strict 40/32/24/18/14/13px hierarchy
  • 96px section padding, whitespace-first
  • 3 sections max, intentional focus
  • Story-driven: hero → breakdown → insights
  • 5 signature colors + semantics

Spacing & Layout

Base Unit: 4px (all spacing snaps to 4-multiples)

Spacing Scale:

  • --spacing-xxs: 4px
  • --spacing-xs: 8px
  • --spacing-sm: 12px
  • --spacing-md: 16px
  • --spacing-lg: 24px (card padding)
  • --spacing-xl: 32px (feature card padding)
  • --spacing-xxl: 48px (signature card padding)
  • --spacing-section: 96px ← Universal vertical rhythm

Border Radius:

  • --radius-xs: 2px (legal buttons)
  • --radius-sm: 6px (inputs)
  • --radius-md: 10px (content cards)
  • --radius-lg: 12px ← CTAs, signature cards
  • --radius-pill: 9999px (pricing only)
  • Every section has 96px top + bottom padding. This is universal.
  • Cards have 24px internal padding (lg) or 32px (xl). Gutters between cards are 24px.
  • Hero sections need breathing room. 96px+ above and below headlines.

How to Implement

  1. Copy design-tokens.css into your project
  2. Use CSS variables: background: var(--sig-forest);
  3. Build components using the patterns above (MetricCard, InsightCard, etc.)
  4. Test responsive at 1024px, 768px, and 375px breakpoints
  5. Apply same tokens to all pages (not just dashboard)
